Project Description

This project focuses on designing and implementing standardized, wizard-based workflows within ArcGIS Tasks. These step-by-step processes will be used to manage and execute cadastral changes and modifications directly within the Israeli National Cadaster dataset, which is maintained using ESRI's Parcel Fabric technology. The goal is to ensure consistency and efficiency in data updates.

Changelog

    • In the Update Attribute step, when a process front does not match to exactly 1 active front, an unmatched fronts layer will be added to the map- helping to visualize the unmatched front on the map..
    • The Stated Areas eraly validtaion check will warn if an incoming temporary parcel does not yet have it's final number (an eralier record should be updated before the current one).
    • The Import or Modify Points tool will output conflicts as red circles around the source point. The radius of the red circle is set by the input search distance.
    • The Evaluation step operations are now grouped by categories, allowing better experience.
    • The Stated Areas eraly validtaion check will handle cases when an incoming active parcel and incoming temporary parcel share the same number.
    • The Start Task Environment step will use the process name by parsing the aprx file name- the user not reqiuerd to type the process name. At that time, users can type other process name for independant cases.
    • Bug fixes regarding to the Evaluation step.
    • Starting from the 2nd step in each task- it is reqiuerd to make sure that a record is activated in order to excecute the steps.
    • First release.
