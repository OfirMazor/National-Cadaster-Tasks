להלן הפעולות לביצוע עדכון מסביבת פיתוח לטסט:
------------------------------------------------------------------------


1. מומלץ: למחוק גרסאות ישנות מה-SDE דרך משתמש sa.

2. מומלץ: להריץ כלי Rebuild Index על כל הטבלאות ב-SDE.

3.  לאתחל את הפרויקט בסביבת פיתוח עם הכלי לאתחול (Reinitialize)

4. לנקות היסטוריית הרצות: Analysis -> History ,בחירת כל הכלים שרצו ולחיצה על X.

5. לייצא את ה-tasks כקובץ בתיקיית Tasks

5. למחוק את כל התיקיות והקבצים בתיקיית TestEnvironment או (ProductionEnvironment) ולהעתיק קבצים חדשים מ-DevelopmentEnvironment.

6. בקובץ configs.py שהועתק- לשנות את המשתנה של הסביבה ל- "Test" או "Production" בהתאם.

7. בפרויקט (aprx), לייבא את ה-tasks מחדש מתיקיית ה-tasks לאחר שהועתקו הקבצים.

8. בכל script-tool שיש ב-tool box של הפרויקט לעדכן את נתיב קובץ ההרצה (הסקריפט) לתיקיית הסקריפטים בטסט במקום התיקייה של הפיתוח.

9. בכל שכבה ב-aprx שהועתק יש לעדכן את נתיב השכבה (כתובת url של ה-featre service) המתאים.

10. לוודא שבאפשרויות של הפרויקט הנתיבים לתיקיות מוגדרים נכון לסביבת הטסט (Project->Options->Current Settings).

11. לוודא שבהגדרות ה-aprx יש הגדרה תקינה של טבלת מעקב אחר האשפים  (TaskHistory).

11. להעלות את הקבצים המעודכנים לגיטהאב.
